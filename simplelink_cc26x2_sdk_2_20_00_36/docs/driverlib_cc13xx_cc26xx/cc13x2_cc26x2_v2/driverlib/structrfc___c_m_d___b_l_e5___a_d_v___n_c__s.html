Detailed Description

Bluetooth 5 Non-Connectable Advertiser Command.

Field Documentation

uint8_t rfc_CMD_BLE5_ADV_NC_s::bEnaCmd

0: No alternative trigger command
1: CMD_TRIGGER can be used as an alternative trigger

uint8_t rfc_CMD_BLE5_ADV_NC_s::bOverride

0: Use default whitening for BLE advertising/data channels
1: Override whitening initialization with value of init

uint8_t rfc_CMD_BLE5_ADV_NC_s::channel

Channel to use
0–39: BLE advertising/data channel index
60–207: Custom frequency; (2300 + channel) MHz
255: Use existing frequency
Others: Reserved

uint8_t rfc_CMD_BLE5_ADV_NC_s::coding

Coding to use for TX if coded PHY is selected. See the Technical Reference Manual for details.

uint16_t rfc_CMD_BLE5_ADV_NC_s::commandNo

The command ID number 0x182D.

struct { ... } rfc_CMD_BLE5_ADV_NC_s::condition
uint8_t rfc_CMD_BLE5_ADV_NC_s::init

If bOverride = 1 or custom frequency is used:
0: Do not use whitening
Other value: Initialization for 7-bit LFSR whitener.

uint8_t rfc_CMD_BLE5_ADV_NC_s::mainMode

PHY to use:
0: 1 Mbps
1: 2 Mbps
2: Coded
3: Reserved

uint8_t rfc_CMD_BLE5_ADV_NC_s::nSkip

Number of skips + 1 if the rule involves skipping. 0: same, 1: next, 2: skip next, ...

uint8_t rfc_CMD_BLE5_ADV_NC_s::pastTrig

0: A trigger in the past is never triggered, or for start of commands, give an error
1: A trigger in the past is triggered as soon as possible

uint32_t rfc_CMD_BLE5_ADV_NC_s::tx20Power

If txPower = 0xFFFF: Transmit power to use for the 20 dBm PA (overrides the one given in radio setup). For other values of txPower, this field is not accessed by the radio CPU and may be omitted from the structure.

uint16_t rfc_CMD_BLE5_ADV_NC_s::txPower

Transmit power to use (overrides the one given in radio setup)
0x0000: Use default TX power
0xFFFF: 20 dBm PA only: Use TX power from tx20Power field (command structure that includes tx20Power must be used)
