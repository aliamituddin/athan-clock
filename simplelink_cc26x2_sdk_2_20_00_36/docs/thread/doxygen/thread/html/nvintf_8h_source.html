29 // Usage Overview
30 //*****************************************************************************
31 /*
32 This file implements a general NV API which can be used with NV drivers
33 designed to conform to this API. The API requires NV items to be described by
34 a three number ID contained in the NVINTF_itemID_t struct. System IDs are also
35 predefined but the user may use their own so long as they do not conflict.
36 Drivers using this interface are also required to return the defined status
37 codes.
38 
39 The particular driver is selected by loading a NVINTF_itemID_t struct with the
40 drivers function ptrs using the drivers public load function. Note that not all
41 of the function pointers are required to be populated so it is important to
42 check for null pointers before making a call. From this point on, NV calls are
43 made through the struct. The first call must be the .initNv() call.
44 If this call is successful, more NV calls can be made. A sample code block is
45 shown below:
46 
47 NVINTF_nvFuncts_t nvFps;
48 NVDRIVER_loadApiPtrs(&nvFps);
49 
50 nvFps.initNV(NULL);
51 // Do some Nv operations
52 nvFps.compactNv(NULL);
53 status = nvFps.readItem(id, 0, len, buf);
54 
55 */
