314  // ----------------------------------------------------------------------------
315  // Property Handlers
316  // ----------------------------------------------------------------------------
317  //
318  // There are 4 types of property handlers for "get", "set", "insert", and
319  // "remove" commands.
320  //
321  // "Get" handlers should get/retrieve the property value and then encode and
322  // write the value into the NCP buffer. If the "get" operation itself fails,
323  // "get" handler should write a `LAST_STATUS` with the error status into the NCP
324  // buffer. The `otError` returned from a "get" handler is the error of writing
325  // into the NCP buffer (e.g., running out buffer), and not of the "get" operation
326  // itself.
327  //
328  // "Set/Insert/Remove" handlers should first decode/parse the value from the
329  // input Spinel frame and then perform the corresponding set/insert/remove
330  // operation. They are not responsible for preparing the Spinel response and
331  // therefore should not write anything to the NCP buffer. The `otError` returned
332  // from a "set/insert/remove" handler indicates the error in either parsing of
333  // the input or the error of set/insert/remove operation.
334  //
335  // The corresponding command handler (e.g., `HandleCommandPropertySet()` for
336  // `VALUE_SET` command) will take care of preparing the Spinel response after
337  // invoking the "set/insert/remove" handler for a given property. For example,
338  // for a `VALUE_SET` command, if the "set" handler returns an error, then a
339  // `LAST_STATUS` update response is prepared, otherwise on success the "get"
340  // handler for the property is used to prepare a `VALUE_IS` Spinel response (in
341  // cases where there is no "get" handler for the property, the input value is
342  // echoed in the response).
343  //
344  // Few properties require special treatment where the response needs to be
345  // prepared directly in the "set" handler (e.g., `HOST_POWER_STATE` or
346  // `NEST_STREAM_MFG`). These properties have a different handler method format
347  // (they expect `aHeader` as an input argument) and are processed separately in
348  // `HandleCommandPropertySet()`.
